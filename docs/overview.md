# RestoX Overview

## วัตถุประสงค์
ระบบจัดการร้านอาหารแบบ CLI สำหรับบทบาทหลัก 5 ฝ่าย:
1. Platform: 
    - สร้างและจัดการร้านอาหารหลายแห่ง (multi-tenant)
    - เก็บค่าสมัครรายเดือน / หัก % จากยอดขาย
    - ดูรายงานรวมทุกสาขา
    - จัดการสิทธิ์เจ้าของร้าน / broadcast ข่าวสาร

2. Store (ร้าน): 
    - จัดการพนักงาน เมนู วัตถุดิบ รายได้-รายจ่าย
    - ตั้งค่าเมนู / สูตรอาหาร / ราคา
    - ตรวจสอบสต๊อกจริง วิเคราะห์ต้นทุน
    - กำหนด loss allowance รายเมนู/วัตถุดิบ

3. Kitchen (ครัว): 
    - รับออเดอร์จากโต๊ะ
    - หักวัตถุดิบตามสูตร โดยอัตโนมัติ
    - สามารถ “แจ้งของเสีย” หรือ “ปรับการใช้” เมื่อจำเป็น
    - ไม่ต้องกรอกข้อมูลเองทุกจาน
    - กดยืนยันการใช้จริงแบบเร็ว (default + แก้ได้)
4. Table (โต๊ะ): 
    - สั่งอาหารเข้าระบบ
    - เช็กสถานะออเดอร์
    - รวม/แยกบิล
    - ปิดออเดอร์
5. Customer (ลูกค้า): 
    - จองโต๊ะ
    - ดูเมนู
    - ให้รีวิว / ให้คะแนนหลังใช้บริการ


## เป้าหมาย
- ฝึกเขียนโปรเจกต์ CLI ที่มีบทบาทหลายฝั่ง
- ใช้ Rust เป็นภาษาหลัก
- มีระบบ REST API สำหรับสื่อสารระหว่าง CLI กับ backend

## เทคโนโลยี
- ภาษา Rust
- REST API (HTTP)
- ฐานข้อมูล SQL/NoSQL (ออกแบบแยกอีกที)
- CLI tools สำหรับแต่ละบทบาท
